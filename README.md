# Chrome 自动刷新扩展

一个功能强大的 Chrome 浏览器扩展，用于自动刷新指定的网页，支持 URL 匹配、事件检测和灵活的定时设置。

![扩展图标](icons/cat.png)

## 功能特性

### 🎯 智能 URL 匹配
- **精确匹配**: 支持完整 URL 匹配
- **前缀匹配**: 支持域名前缀匹配（如 `example.com` 匹配 `example.com/path`）
- **子路径匹配**: 支持子路径匹配
- **批量管理**: 支持添加、删除、清空 URL 列表

### ⏰ 灵活定时刷新
- **时间单位**: 支持秒和分钟两种时间单位
- **自定义间隔**: 可设置 1 秒到 24 小时之间的任意时间间隔
- **智能重置**: 检测到用户活动时自动重置计时器

### 🖱️ 事件检测与重置
- **鼠标移动**: 检测鼠标移动事件
- **鼠标单击**: 检测鼠标点击事件
- **鼠标双击**: 检测鼠标双击事件
- **鼠标滚轮**: 检测鼠标滚轮滚动事件
- **自动重置**: 检测到用户活动时，计时器自动重置

### 🎨 用户友好界面
- **简洁设置界面**: 直观的弹出窗口配置界面
- **一键启用/禁用**: 快速切换扩展状态
- **实时状态显示**: 清晰的状态指示器
- **批量操作**: 支持批量添加和删除 URL

### 🔒 安全可靠
- **URL 验证**: 自动验证和标准化输入的 URL
- **安全存储**: 使用 Chrome 本地存储安全保存配置
- **错误处理**: 完善的错误处理和日志记录
- **资源清理**: 页面卸载时自动清理事件监听器

## 安装方法

### 从源码安装
1. 克隆或下载本仓库到本地
2. 打开 Chrome 浏览器，进入 `chrome://extensions/`
3. 开启右上角的"开发者模式"
4. 点击"加载已解压的扩展程序"
5. 选择项目根目录（包含 `manifest.json` 的文件夹）
6. 扩展将自动加载并显示在扩展列表中

### 使用说明
1. **添加 URL**: 在扩展弹出窗口中输入要监控的 URL（支持完整 URL 或域名）
2. **设置时间**: 选择刷新间隔时间（秒或分钟）
3. **选择事件**: 勾选需要监听的事件类型
4. **启用扩展**: 点击右上角的切换按钮启用自动刷新
5. **保存设置**: 点击"保存设置"按钮应用配置

## 核心功能详解

### 1. URL 匹配机制
扩展使用智能 URL 匹配算法：
- **标准化处理**: 自动添加协议、转换为小写、移除尾部斜杠
- **多层匹配**: 支持精确匹配、前缀匹配和子路径匹配
- **安全验证**: 验证域名格式和 IP 地址格式

### 2. 计时器管理
- **精确计时**: 使用 JavaScript `setTimeout` 实现精确计时
- **事件重置**: 检测到用户活动时自动重置计时器
- **状态同步**: 所有标签页状态同步更新

### 3. 配置持久化
- **本地存储**: 使用 `chrome.storage.local` 保存配置
- **实时同步**: 配置更改后实时应用到所有标签页
- **默认值**: 提供合理的默认配置值

## 配置选项

### URL 列表管理
- 支持添加多个监控 URL
- 支持删除单个 URL
- 支持清空整个 URL 列表
- 支持 URL 相似性检测和合并

### 刷新时间设置
- **时间范围**: 1 秒 - 3600 秒 或 1 分钟 - 1440 分钟
- **单位切换**: 秒和分钟两种时间单位
- **默认值**: 15 分钟

### 事件监听设置
- **鼠标移动** (`mousemove`): 检测鼠标移动
- **鼠标单击** (`click`): 检测鼠标点击
- **鼠标双击** (`dblclick`): 检测鼠标双击
- **鼠标滚轮** (`wheel`): 检测鼠标滚动

## 技术实现

### 架构设计
1. **内容脚本** (`app.js`): 注入到匹配的网页中，负责计时和刷新
2. **后台脚本** (`background.js`): 管理扩展状态和消息传递
3. **弹出窗口** (`popup/`): 提供用户配置界面
4. **工具模块** (`utils.js`): 提供通用的工具函数

### 消息通信
- `chrome.runtime.sendMessage`: 弹出窗口到后台脚本
- `chrome.runtime.onMessage`: 后台脚本到内容脚本
- `chrome.tabs.sendMessage`: 后台脚本到特定标签页

### 存储管理
- `chrome.storage.local.get`: 读取配置
- `chrome.storage.local.set`: 保存配置
- `chrome.storage.local.remove`: 删除配置

## 常见问题

### Q: 扩展为什么没有反应？
A: 请检查：
1. 扩展是否已启用（图标显示为启动状态）
2. 当前页面 URL 是否在监控列表中
3. 刷新时间设置是否合理

### Q: 如何添加多个 URL？
A: 在 URL 输入框中输入一个 URL，点击"添加 URL"按钮，重复此过程添加多个 URL。

### Q: 事件检测有什么作用？
A: 当检测到用户活动（如鼠标移动、点击等）时，计时器会重置，避免在用户操作时刷新页面。

### Q: 扩展会影响浏览器性能吗？
A: 扩展设计轻量，只在匹配的页面中运行计时器，对性能影响极小。
